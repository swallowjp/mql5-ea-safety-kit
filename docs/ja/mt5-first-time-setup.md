# はじめての MT5 EA セットアップガイド（LotSafety 診断スクリプト）

このガイドは、MetaTrader 5（MT5）で Expert Advisor（EA）やスクリプトを初めて扱う方向けに、既存の `LotSafety` 診断スクリプトをインストールして確認する手順を説明します。

`LotSafety` はロット数（注文数量）の安全確認を支援するための部品です。このガイドで使用する `TestLotSafety.mq5` は診断用スクリプトであり、売買シグナル、エントリー条件、決済条件、利益を出すためのロジックは含みません。

> 注意: このプロジェクトは金融助言、投資助言、利益保証を提供しません。実口座で EA を使う前に、必ずご自身の MT5 環境、ブローカー、口座条件、対象シンボルで確認してください。

## このガイドで行うこと

このガイドでは、次の 2 つのファイルを MT5 の正しいデータフォルダへ配置し、MetaEditor でコンパイルして、チャート上で診断結果を確認します。

| リポジトリ内のファイル | MT5 データフォルダ内の配置先 | 目的 |
| --- | --- | --- |
| `MQL5/Include/LotSafety.mqh` | `MQL5/Include/LotSafety.mqh` | ロット安全確認の include ファイル |
| `MQL5/Scripts/TestLotSafety.mq5` | `MQL5/Scripts/TestLotSafety.mq5` | 診断結果を Experts ログへ出力するスクリプト |

このスクリプトは、現在のチャートのシンボル（例: `EURUSD`、`USDJPY`、`XAUUSD` など）について、ブローカーが提供する最小ロット、最大ロット、ロット刻みを読み取り、サンプル値の検証結果をログへ出力します。

## このガイドで行わないこと

このガイドでは、次の操作は行いません。

* 注文を出す
* 注文を変更する
* 注文を決済する
* 売買シグナルやトレード戦略を追加する
* 口座残高、証拠金、余剰証拠金、損益をもとにした資金管理ルールを実装する

`TestLotSafety.mq5` は診断ログを出すためのスクリプトです。注文送信、注文変更、注文決済の処理は含まれていません。

## 前提条件

* MetaTrader 5 がインストール済みであること。
* MT5 にログイン済みで、少なくとも 1 つのチャートを開けること。
* このリポジトリのファイルをダウンロード済み、または Git で取得済みであること。
* Windows のエクスプローラーでファイルをコピーできること。

この手順では、MetaEditor のコンパイル結果と MT5 の Experts ログを確認します。MetaEditor でコンパイルできることと、MT5 のチャート上でスクリプトを実行してログを確認できることは別の確認です。

## 1. MT5 のデータフォルダを開く

MT5 では、プログラム本体のインストール先とは別に「データフォルダ」があります。MQL5 ファイルは、このデータフォルダ内へ配置する必要があります。

1. MT5 を起動します。
2. 上部メニューから **ファイル** を選びます。
3. **データフォルダを開く** を選びます。
4. Windows のエクスプローラーで MT5 のデータフォルダが開きます。
5. その中に `MQL5` フォルダがあることを確認します。

重要: MT5 を複数インストールしている場合、データフォルダも複数存在することがあります。実際に使用する MT5 から **ファイル > データフォルダを開く** を選び、そのフォルダへ配置してください。

## 2. `LotSafety.mqh` を Include フォルダへ配置する

1. MT5 のデータフォルダ内で `MQL5/Include` フォルダを開きます。
2. リポジトリ内の `MQL5/Include/LotSafety.mqh` をコピーします。
3. MT5 データフォルダ内の `MQL5/Include/LotSafety.mqh` として配置します。

配置後の例:

```text
<MT5 データフォルダ>/MQL5/Include/LotSafety.mqh
```

`LotSafety.mqh` は include ファイルです。単独でチャートへドラッグして実行するものではありません。`TestLotSafety.mq5` から読み込まれます。

## 3. `TestLotSafety.mq5` を Scripts フォルダへ配置する

1. MT5 のデータフォルダ内で `MQL5/Scripts` フォルダを開きます。
2. リポジトリ内の `MQL5/Scripts/TestLotSafety.mq5` をコピーします。
3. MT5 データフォルダ内の `MQL5/Scripts/TestLotSafety.mq5` として配置します。

配置後の例:

```text
<MT5 データフォルダ>/MQL5/Scripts/TestLotSafety.mq5
```

ファイル名は `TestLotSafety.mq5` のままにしてください。ブラウザーやエクスプローラーでコピーしたときに `TestLotSafety(2).mq5` や `TestLotSafety - コピー.mq5` のような名前になると、ナビゲータやコンパイル結果の確認時に混乱しやすくなります。

## 4. MT5 から MetaEditor を開く

MetaEditor は MQL5 ファイルを編集・コンパイルするためのツールです。

1. MT5 を開きます。
2. キーボードの **F4** を押します。
3. または、上部メニューから **ツール > MetaQuotes Language Editor** を選びます。
4. MetaEditor が開いたら、左側のナビゲータで `Scripts` または `Include` を確認します。

MetaEditor が別ウィンドウで開きます。MT5 本体と MetaEditor は別の画面ですが、同じデータフォルダ内の MQL5 ファイルを扱います。

## 5. `TestLotSafety.mq5` を開いてコンパイルする

1. MetaEditor の左側ナビゲータで `Scripts` を開きます。
2. `TestLotSafety.mq5` を開きます。
3. 上部ツールバーの **コンパイル** を押します。
4. または、キーボードの **F7** を押します。
5. 画面下部の Toolbox または Errors に表示されるコンパイル結果を確認します。

期待する結果は次のような内容です。

```text
0 errors, 0 warnings
```

`0 errors, 0 warnings` は、MetaEditor がソースコードをコンパイルでき、エラーと警告を検出しなかったことを意味します。ただし、これは「実際のブローカーシンボルでの診断実行が完了した」ことや「今後の運用が安全である」ことを意味しません。次の手順で、MT5 のチャート上でスクリプトを実行し、Experts ログを確認してください。

## 6. MT5 のナビゲータを更新する

コンパイル後、MT5 本体に戻ります。

1. MT5 の左側にある **ナビゲータ** を表示します。
   * 表示されていない場合は、上部メニューから **表示 > ナビゲータ** を選びます。
2. ナビゲータ内の **スクリプト** を探します。
3. ナビゲータ上で右クリックし、**更新** を選びます。
4. `TestLotSafety` が表示されることを確認します。

表示されない場合は、配置先が `MQL5/Scripts` になっているか、コンパイルで `.ex5` ファイルが作成されているか、実際に使用中の MT5 データフォルダへ配置したかを確認してください。

## 7. チャート上で `TestLotSafety` を実行する

1. MT5 で確認したいシンボルのチャートを開きます。
   * 例: `EURUSD`、`USDJPY`、`XAUUSD` など。
2. ナビゲータの **スクリプト** から `TestLotSafety` を探します。
3. `TestLotSafety` をチャートへドラッグ＆ドロップします。
4. 実行確認画面が表示された場合は、内容を確認して実行します。

このスクリプトは現在のチャートの `_Symbol` を使います。たとえば `USDJPY` のチャートで実行すると、`USDJPY` の最小ロット、最大ロット、ロット刻みを読み取って診断します。

## 8. Toolbox / Experts ログを開く

診断結果はチャート上ではなく、MT5 の Toolbox / Experts ログに出力されます。

1. MT5 の下部にある **ツールボックス** を開きます。
   * 表示されていない場合は、上部メニューから **表示 > ツールボックス** を選びます。
2. **エキスパート** または **Experts** タブを選びます。
3. `TestLotSafety` や `LotSafety` を含むログ行を確認します。

ログには、対象シンボル、最小ロット、最大ロット、ロット刻み、要求ロット、正規化後ロット、成功または失敗の理由が表示されます。

## 9. サマリー行の意味

診断が成功すると、Experts ログに次のようなサマリー行が表示されます。

```text
LotSafety test summary: total=23 passed=23 failed=0
```

この意味は次のとおりです。

* `total=23`: 診断ケースの合計数が 23 件です。
* `passed=23`: 期待どおりに通過した診断ケースが 23 件です。
* `failed=0`: 期待と異なる結果になった診断ケースが 0 件です。

このサマリーは、`LotSafety` のロット正規化ロジックと診断ケースが、その MT5 環境で期待どおりに実行されたことを示します。ただし、利益が出ること、損失が出ないこと、ブローカーのすべてのシンボルで同じ結果になることを保証するものではありません。

## 10. なぜ注文を出さないのか

`TestLotSafety.mq5` は、ロット数の検証結果をログへ出力するための診断スクリプトです。注文に関する操作は行いません。

このスクリプトには、次のような注文操作は含まれていません。

* 成行注文や指値注文を送信する処理
* 既存ポジションを変更する処理
* ストップロスやテイクプロフィットを変更する処理
* ポジションを決済する処理

確認対象は、ブローカーが提供するシンボル仕様と、サンプルの要求ロットが安全に拒否または下方向へ正規化されるかどうかです。

## 11. よくある間違いと確認ポイント

### 違う MT5 データフォルダへ配置している

MT5 を複数インストールしている場合、別のターミナルのデータフォルダへコピーしてしまうことがあります。必ず、実行する MT5 から **ファイル > データフォルダを開く** を選んでください。

### `TestLotSafety(2).mq5` のような名前になっている

同じファイルを複数回ダウンロードすると、ファイル名に `(2)` などが付く場合があります。初心者の方は、まず `TestLotSafety.mq5` という名前にそろえると確認しやすくなります。

### 古い `.ex5` ファイルを実行している

MetaEditor でコンパイルすると `.mq5` から `.ex5` が作成されます。古い `.ex5` が残っていると、意図したソースコードと違う実行ファイルを見ている可能性があります。必要に応じて `TestLotSafety.ex5` を削除してから、再度 `TestLotSafety.mq5` をコンパイルしてください。

### `LotSafety.mqh` を Scripts に置いている

`LotSafety.mqh` は `MQL5/Include` に配置します。`MQL5/Scripts` に置くのは `TestLotSafety.mq5` です。

### `TestLotSafety.mq5` を Include に置いている

`TestLotSafety.mq5` はスクリプトなので `MQL5/Scripts` に配置します。`MQL5/Include` に置くと、MT5 のナビゲータでスクリプトとして見つけにくくなります。

### コンパイルだけで確認が完了したと思っている

MetaEditor の `0 errors, 0 warnings` はコンパイル確認です。MT5 のチャート上で実行し、Experts ログにサマリー行が出ることも確認してください。

### Experts ログではなく Journal だけを見ている

MT5 には複数のログタブがあります。`TestLotSafety` の診断結果は **エキスパート** または **Experts** タブで確認してください。

## 12. 任意: DrawdownGuard 診断スクリプトを確認する

このリポジトリには、ロット安全確認とは別に、日次ドローダウンと総ドローダウンを評価する `DrawdownGuard` も含まれています。`DrawdownGuard` は、現在の有効証拠金、日次開始時の有効証拠金、口座開始時の有効証拠金、日次最大損失額、総最大損失額を、呼び出し側が渡した同じ口座価値単位で比較します。JPY（日本円）や USD（米ドル）などの通貨換算は行わないため、すべての値を同じ口座通貨または同じ口座価値単位で渡す責任は利用者側にあります。

`DrawdownGuard` は判定結果を返すだけの安全確認部品です。特定のプロップファーム規則を実装するものではなく、売買シグナル、注文送信、注文変更、ポジション決済、利益を保証するロジックは含みません。`trading_allowed` が `false` になった場合に EA を停止する、発注前チェックで拒否する、通知だけ出す、などの対応は利用者自身の EA 側で決める必要があります。

確認する場合は、次の 2 つのファイルを MT5 データフォルダへ配置します。

| リポジトリ内のファイル | MT5 データフォルダ内の配置先 | 役割 |
| --- | --- | --- |
| `MQL5/Include/DrawdownGuard.mqh` | `MQL5/Include/DrawdownGuard.mqh` | ドローダウン判定の include ファイル |
| `MQL5/Scripts/TestDrawdownGuard.mq5` | `MQL5/Scripts/TestDrawdownGuard.mq5` | 固定値の診断結果を Experts ログへ出力するスクリプト |

`TestDrawdownGuard.mq5` は、注文を出さない固定値の診断スクリプトです。ドローダウンなし、日次損失が上限未満・上限ちょうど・上限超過、総損失が上限未満・上限ちょうど・上限超過、両方の上限超過、負の有効証拠金、負の損失上限、現在の有効証拠金が開始時の有効証拠金を上回る不整合ケースを、ハードコードされた期待値で確認します。

MetaEditor で `TestDrawdownGuard.mq5` をコンパイルし、任意のチャートで実行すると、Experts ログに次のようなサマリー行が表示されます。

```text
DrawdownGuard test summary: total=13 passed=13 failed=0
```

このサマリーは、固定値の診断ケースが期待どおりに実行されたことを示します。ただし、損失を防ぐこと、利益が出ること、特定のルールに合格すること、すべてのブローカー環境で同じ運用判断になることを保証するものではありません。日次の開始時刻についても、ブローカーサーバー時刻、VPS（Virtual Private Server）時刻、UTC（協定世界時）、JST（日本標準時）を同じものとして扱わず、利用者自身が明示的に決めて記録してください。

## 13. 確認記録に残すとよい項目

監査性と再現性のため、確認した内容をメモしておくことを推奨します。

* 確認日
* MT5 のビルド番号
* ブローカー名（公開してよい範囲）
* 口座種別（デモ口座、検証用口座など。口座番号は記録しない）
* 対象シンボル
* MetaEditor のコンパイル結果
* Experts ログのサマリー行
* エラーや警告があった場合の内容

口座番号、ログイン ID、パスワード、API キー、個人の取引履歴などの秘密情報は、このリポジトリや公開 Issue に貼り付けないでください。

## 14. 次のステップ

`TestLotSafety` の診断が期待どおりに完了したら、次の点を確認してください。

* 使用予定シンボルごとに最小ロット、最大ロット、ロット刻みが異なる可能性があること。
* デモ口座、検証用口座、実口座でシンボル仕様が異なる可能性があること。
* コンパイル確認、チャート上での実行確認、Experts ログ確認はそれぞれ別の確認であること。
* この診断はロット安全確認の一部であり、日次ドローダウン、総ドローダウン、口座通貨換算、サーバー時刻、UTC（協定世界時）、JST（日本標準時）などのリスク管理を代替しないこと。
